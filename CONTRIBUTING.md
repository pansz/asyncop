# Contributing to AsyncOp

Thank you for your interest in contributing to AsyncOp! Here are some guidelines to help you get started.

## Development Setup

1. Clone the repository
2. Install dependencies (GLib/Qt development packages, spdlog)
3. Create a build directory and run cmake
4. Run tests to verify your setup

## Code Style

- Follow Google C++ Style Guide
- Use C++17 features where appropriate
- Maintain consistency with existing code
- Include documentation for public APIs
- Add tests for new functionality

## Pull Request Process

1. Create a feature branch from `main`
2. Add your changes with appropriate tests
3. Update documentation if needed
4. Submit a pull request with a clear description
5. Address review comments as needed

## Testing

- Add unit tests for new functionality
- Run existing tests to ensure no regressions
- Test both Qt and GLib backends if applicable

## Questions?

If you have any questions, feel free to open an issue.
